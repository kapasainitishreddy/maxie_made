The philosophy

Why SRB exists.

The studio name is also the why. Three names. One company. Built so the people behind it have a reason to keep going.

01

Ship > Perfect

Every product on this page is a v1 that was ugly for six months before it was useful. The default is to ship. The default is to talk to users. The default is to delete code that doesn't earn its place.

02

Open by default

The whole monorepo is MIT-licensed on GitHub. If you want to fork, deploy, or learn from it, do. The only thing we charge for is the hosted version, and only because servers cost money.

03

One person, many verticals

Five niches in one studio sounds reckless. It is. It also means cross-pollination: the same Stripe billing pattern, the same auth flow, the same deploy story — built once, used five times.

04

$0 to launch

Free tiers of Netlify, Fly.io, Neon, Clerk, Stripe, GitHub Actions. We prove the business model works before spending a dollar. The cloud bill for the first 100 users is under $5/mo.

05

Build in public

Every commit, every MRR update, every embarrassing bug — posted on socials. The good, the bad, the receipts. The point isn't the metrics. The point is the documentation of a life spent shipping.

06

Family first

SRB is Sisira, Rathnakar, Bhagya. The studio exists so that one day, the work pays for the people it was named after. Everything else is a means to that end.

The story

One engineer. Five products. A studio named after family.

SRB started as a folder name on a laptop. Then it became five production-grade SaaS products, each with billing, auth, CI/CD, and paying customers — or, more often, with a real plan to find them.

The studio name isn't a brand exercise. It's three first names. The sister, the father, the mother. Everything in the company is a quiet promise to them that the work is going somewhere.

What you see on this site is the public version of that work. The code is open. The decisions are documented. The bugs are confessed. Welcome to the build.
